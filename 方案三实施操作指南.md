## 🚀 部署操作步骤（完成准备后执行）

### 步骤 1：创建 GitHub 仓库

1. **登录 GitHub**
   - 访问：https://github.com

   - 点击 "Sign in"

2. **创建新仓库**
   - 点击右上角 "+" 图标
   - 选择 "New repository"

3. **填写仓库信息**
   ```

   Description: 天津菜点餐系统
   Public: ✅ (必须选择 Public)
   Initialize this repository with a README: ❌ (不要勾选)
   ```

4. **点击 "Create repository"**

**结果**：
- ✅ 仓库创建成功
- ✅ 看到仓库页面
- ✅ 记下仓库地址：`https://github.com/您的用户名/tianjin-menu`

---

### 步骤 2：初始化本地 Git 仓库

1. **打开项目目录**
   ```
   D:\卫宁文件\AI\LBY\CD
   ```

2. **右键选择 "Git Bash Here"**

3. **初始化仓库**
   ```bash
   git init
   ```

   **预期输出**：
   ```
   Initialized empty Git repository in D:/卫宁文件/AI/LBY/CD/.git/
   ```

4. **添加所有文件**
   ```bash

   ```

   **说明**：
   - 添加所有文件到 Git

5. **提交更改**
   ```bash

   ```

   **预期输出**：
   ```
   1 file changed, X insertions(+)
   create mode 100644 index.html
   create mode 100644 style.css
   create mode 100644 script.js
   ```

---

### 步骤 3：关联远程仓库

1. **获取仓库地址**
   - 在 GitHub 仓库页面
   - 点击 "Code" 按钮
   - 复制 HTTPS 地址
   - 格式：`https://github.com/您的用户名/tianjin-menu.git`

2. **关联远程仓库**
   ```bash
   git remote add origin https://github.com/您的用户名/tianjin-menu.git
   ```

   **示例**：
   ```bash
   git remote add origin https://github.com/zhangsan/tianjin-menu.git
   ```

3. **验证关联**
   ```bash
   git remote -v
   ```

   **预期输出**：
   ```
   origin  https://github.com/您的用户名/tianjin-menu.git (fetch)
   origin  https://github.com/您的用户名/tianjin-menu.git (push)
   ```

---

### 步骤 4：推送到 GitHub

1. **推送代码**
   ```bash
   git push -u origin main
   ```

   **或者**（如果分支名是 master）：
   ```bash
   git push -u origin master
   ```

2. **输入 GitHub 账号信息**
   - Username: 您的 GitHub 用户名
   - Password: 您的 GitHub 密码（或个人访问令牌）

   **注意**：
   - GitHub 现在要求使用个人访问令牌（Token）而非密码
   - 如果密码验证失败，需要创建 Token

3. **创建个人访问令牌（如果需要）**
   - 访问：https://github.com/settings/tokens
   - 点击 "Generate new token"
   - 选择 "Generate new token (classic)"
   - 勾选 "repo" 权限
   - 点击 "Generate token"
   - **复制并保存 Token**（只显示一次）
   - 使用 Token 代替密码

4. **推送成功**
   
   **预期输出**：
   ```
   Enumerating objects: X, done.
   Counting objects: 100% (X/X), done.
   Delta compression using up to X threads
   Compressing objects: 100% (X/X), done.
   Writing objects: 100% (X/X), X KiB | X MiB/s, done.
   Total X (delta 0), reused 0 (delta 0), pack-reused 0
   remote: Create a pull request for 'main' on GitHub by visiting:
        https://github.com/您的用户名/tianjin-menu/pulls
   To https://github.com/您的用户名/tianjin-menu.git
    * [new branch]      main -> main
   Branch 'main' set up to track remote branch 'main' from 'origin'.
   ```

---

### 步骤 5：启用 GitHub Pages

1. **进入仓库设置**
   - 在 GitHub 仓库页面
   - 点击 "Settings" 选项卡

2. **找到 Pages 选项**
   - 左侧菜单向下滚动
   - 找到并点击 "Pages"

3. **配置 Pages 设置**
   ```
   Source: Deploy from a branch
   Branch: 选择 "main" 或 "master"
   Folder: 选择 "/ (root)"
   ```

4. **点击 "Save"**

5. **等待部署**
   - 页面顶部会显示 "Your site is live at..."
   - 等待 1-2 分钟

## ❓ 常见问题

### Q1: 推送时提示认证失败？

**A**: 需要使用个人访问令牌（Token）而非密码
- 访问：https://github.com/settings/tokens
- 创建新 Token
- 使用 Token 代替密码

### Q2: 推送后看不到网站？

**A**: 
1. 等待 1-2 分钟
2. 检查 Pages 设置是否正确
3. 刷新页面

### Q3: 访问速度慢？

**A**: 
- GitHub Pages 在国内访问可能较慢
- 可以考虑使用 Vercel 或 Netlify
- 或使用 CDN 加速

### Q4: 网站地址可以自定义吗？

**A**: 可以
- 购买域名
- 配置 DNS
- 在 GitHub Pages 设置中绑定域名
