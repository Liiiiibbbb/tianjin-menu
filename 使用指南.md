## 📱 手机访问（同一 WiFi）

### 方法一：自动启动（推荐）

1. **双击运行** `启动服务器.bat`
2. **查看显示的地址**：
   ```
   手机访问：http://192.168.31.102:8000
   ```
3. **手机操作**：
   - 连接同一 WiFi
   - 打开浏览器
   - 输入显示的地址
   - 访问成功！

### 方法二：手动启动

1. **查看 IP 地址**
   - 双击：`查看 IP 地址.bat`
   - 记下显示的 IP 地址

2. **启动服务器**
   - 双击：`启动服务器.bat`

3. **手机访问**
   - 地址格式：`http://[IP 地址]:8000`
   - 例如：`http://192.168.31.102:8000`

## 🌐 远程访问（任何地方）

### 方案一：GitHub Pages（免费推荐）

**步骤**：
1. 注册 GitHub 账号
2. 创建仓库
3. 上传项目文件
4. 启用 GitHub Pages
5. 获得公网地址

**优点**：
- ✅ 完全免费
- ✅ 全球可访问
- ✅ 永久有效
- ✅ 稳定可靠

**详细教程**：查看 `部署到 GitHub Pages.md`

### 方案二：Vercel（免费）

**网址**：https://vercel.com

**步骤**：
1. 注册 Vercel 账号
2. 导入 GitHub 仓库
3. 自动部署
4. 获得公网地址

**优点**：
- ✅ 国内访问更快
- ✅ 自动 HTTPS
- ✅ 全球 CDN

### 方案三：ngrok（临时测试）

**适用场景**：临时展示、测试

**步骤**：
1. 下载 ngrok
2. 注册获取 token
3. 启动：`ngrok http 8000`
4. 获得临时公网地址

**缺点**：
- ❌ 每次重启地址变化
- ❌ 速度较慢
- ❌ 不稳定

## ⚠️ 重要提示

### 手机访问必须满足

1. ✅ **同一 WiFi**
   - 手机和电脑必须连接同一 WiFi
   - 不能一个 WiFi 一个流量

2. ✅ **服务器运行**
   - 必须运行 `启动服务器.bat`
   - 或手动启动 Python 服务器

3. ✅ **防火墙允许**
   - Windows 防火墙可能阻止
   - 临时关闭或添加例外

### 部署到公网注意

1. ✅ **只上传必要文件**
   - 上传：HTML、CSS、JS
   - 不上传：.bat 文件、文档

2. ✅ **检查功能**
   - localStorage 功能正常
   - 无服务器端代码
   - 纯静态网站

3. ✅ **测试访问**
   - 手机测试
   - 电脑测试
   - 不同浏览器测试

## 🔧 故障排查

### 问题 1：双击 index.html 无法使用

**可能原因**：
- 浏览器不兼容
- 文件损坏

**解决方案**：
1. 更换浏览器（Chrome/Firefox/Edge）
2. 重新下载项目文件

---

### 问题 2：启动服务器失败

**错误信息**：
```
ModuleNotFoundError: No module named 'http.server'
```

**原因**：Python 版本问题

**解决方案**：
```cmd
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```

---

### 问题 3：手机无法访问

**排查步骤**：

1. **检查 WiFi**
   ```
   电脑：连接 WiFi A
   手机：连接 WiFi A（不能是 WiFi B 或流量）
   ```

2. **检查 IP 地址**
   ```
   双击：查看 IP 地址.bat
   确认输入的 IP 正确
   ```

3. **检查防火墙**
   ```
   控制面板 → Windows Defender 防火墙
   → 关闭防火墙（临时）
   ```

4. **检查端口**
   ```
   确保 8000 端口未被占用
   可以更换为 8080、8888 等
   ```

---

### 问题 4：GitHub Pages 部署失败

**常见问题**：

1. **文件未上传**
   - 确保上传了所有必要文件
   - 检查文件名大小写

2. **Pages 未启用**
   - 进入 Settings → Pages
   - 确认 Source 配置正确

3. **等待时间不够**
   - 部署需要 1-2 分钟
   - 耐心等待再刷新
